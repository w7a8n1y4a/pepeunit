## 3. Что такое Pepeunit

*[На экране: чистая схема — Git-репозиторий → Pepeunit → Устройства]*

> Pepeunit — Берёт код из Git. Превращает его в готовое устройство. Без боли.

> Главная идея проста. Тот, кто пишет код — Разработчик. Тот, кто пользуется устройством — Пользователь. Они не обязаны быть одним человеком.

*[На экране: схема разделения Developer / User]*
*[На экране: через аишку как гендальф даёт фродо колечко]*

> Разработчик пишет код в git репозиторий. Пользователь нажимает кнопки в интерфейсе. Всё.

## 4. Два репозитория

*[На экране: GitLab / GitHub, два репозитория]*

> Для нашего примера — стриминг видео на OLED-экраны — нужно два репозитория. Серверный на Python. И IoT на MicroPython для ESP32.

*[На экране: содержимое репозитория, выделены два файла]*

> В каждом репозитории лежат два файла. Запомните их.

> `env_example.json` — шаблон настроек. Wi-Fi, пины, адреса. Всё, что делает ваше устройство уникальным.

> `schema_example.json` — описание входов и выходов. Что устройство принимает, что отдаёт. По этому файлу Pepeunit создаёт MQTT-топики.

*[На экране: простая анимация — два файла превращаются в работающий Unit]*

> Разработчик написал эти два файла. Pepeunit сделает Unit индивидуальным. Пользователь завершит настройку.

## 7. Связываем Unit-ы

*[На экране: интерфейс UnitNode, стрелки Output → Input]*

> Unit-ы созданы, но они пока не знают друг о друге. Как соседи в многоэтажке.

> Открываем schema — видим входы и выходы. Output серверного Unit ставим в Input экранного. Готово. Теперь они общаются.

*[На экране: заполнение env]*

> Дальше заполняем env — Wi-Fi сеть, номера пинов. Всё что написано в документации репозитория.

## 8. Заливаем и запускаем

*[На экране: терминал, esptool, ampy]*

> Скачиваем архив из Pepeunit. На ESP заливаем через esptool и ampy. На сервер — uv и запускаем Python-скрипт.

> Три команды. Не двадцать три. Три.

## 11. Что ещё умеет Pepeunit

*[На экране: быстрая нарезка интерфейсов]*

> Мы посмотрели на Pepeunit глазами Пользователя. Но это не всё.

> DataPipe — конвейеры данных. Пишете YML — получаете обработку телеметрии.

*[На экране: Grafana дашборд]*

> Автоматические обновления из Git. Реестр репозиториев. Система прав доступа.

> И всё это не привязано к языку программирования или платформе.
