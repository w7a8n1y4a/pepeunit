# Система сравнения инстансов

Задача системы - дать пользователям место на каждом инстансе, где они могут понять состояние сети Pepeunit

## Единый сервис парсер

1. Сервис опражшивает все инстансы которые он знает о их состоянии
1. Сервис обходит все досупные инстансы за 1 час, равномерно распределяя запросы
1. По ip определяется геолокация, ping, и тд
1. Элементы добавляются вручную
1. Данные из сервиса парсера, кэшируются 1 раз в час

## Генерация данных для парсера
1. Ручка domain.com/pepeunit: Отдача состояния системы с настройками, происходит через REST и GQL запрос с кешом и обновлением раз в 1 час
1. Ручка gql и rest отдающая список доменов, которые есть у инстанса с пагинацией конечно же

Данные включают:
1. Базовые фича флаги инстанса
1. Базовые настройки инстанса
1. Когда в последний раз прошли интеграционники и результат
1. Статистики инстанса
1. Контакты владельца инстанса
1. Убрать автора, стало избыточной информацией

## Админ сможет

1. Вызов интеграционных тестов с фронтенда, пароли все при этом генерируются уже рандомно, в целом тесты перестроены, чтобы вызываться без проверки функционала приватных репозиториев, через спец флаг
1. Кнопка принудительного сканирования внешних инстансов, проходит по всем доменам отдельно и уточняет информацию
1. Кнопка узнающая у других инстансов, о их списке доменов, позволяет синхронизировать списки доменов у разных инстансов

## Реализация

Нужна таблица instances для хранения всех доступных инстансов:
- Домен инстанса
- ip адрес инстансов
- ping
- информация о геолокации
- Время сбора информации
- Накопительный каунтер, числа положительных сборов данных
- Статус последнего сбора информации
- Ошибка сбора если есть
- хранение json ответов целиком, работать будем по версии схем, чтобы добиться универсальности между разными инстансами. Каждая последующая версия, будет знать о нюансах предыдущих

1. Потребуется полностью перестроить концепцию интеграционных тестов
1. Нужно сделать факт чекинг конфига при запуске, по некоторым переменным, например контактам owner
