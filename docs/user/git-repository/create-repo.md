# Создание Repo

:::info Где найти?
Есть два способа:
1. На главной странице инстанса, найдите в верхнем правом углу кнопку-картинку - `Микросхема`
1. На главной странице инстанса, найдите в верхнем левом углу кнопку поиска. Выберите сущность `Registry` и найдите при помощи фильтров нужную вам сущность. Нажмите кнопку `Pick`. Далее достаточно нажать кнопку `Create Repo`
:::

## Выбор RepositoryRegisty

Достаточно нажать на `Pick` в модельном окне поиска. Если нужный вам [RepositoryRegisty](/definitions#repositoryregistry) отсутствует, можно создать новый, нажав на кнопку `Create Registry`.

:::danger
Устанавливается один раз в момент создания
:::

## Целевая Ветка

`Целевая ветка` - [Ветка](/definitions#git-branch) [Git](/definitions#git) репозитория по умолчанию, предназначенная для корректной работы обновлений и создания [Unit](/definitions#unit). Обычно это `master`, `main` или `release`.
 
:::info
Можно изменить в будущем
:::

## Имя вашего Repo

Требования:
1. Уникальность на текущем инстансе
1. Длинна от `4` до `20` символов
1. Разрешённые символы: `a-z, A-Z, 0-9 и _.-`

:::info
Можно изменить в будущем
:::

## Компилируемость
Обычно значение данного флага можно подсмотреть в [Readme](/definitions#readme-md) [Git](/definitions#git) репозитория.

Установите флаг `True` - для [Компилируемых](/definitions#compilable) репозиториев и `False` для [Интерпретируемых](/definitions#interpretable).

:::warning
От значения данного флага будет зависеть [первичное получение файлов](/user/unit/create-physic-unit#получение-фаилов-развертывания) для [Unit](/definitions#unit)
:::

:::info
Можно изменить в будущем
:::
